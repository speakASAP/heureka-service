## Purpose

Control changes to intent, architecture, contracts, and deployment behavior.

## Controlled Changes

Vision/constitution changes require human approval; major architecture changes require ADRs; contract changes require validation plans; sensitive-data changes require policy review.

## Evidence

Controlled changes must link task, plan, goal impact, validation report, and gate output.
